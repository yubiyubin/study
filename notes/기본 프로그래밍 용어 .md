# 기본 프로그래밍 용어

## 프로그래밍 기본

- API (Application Programming Interface)- 앱끼리 통신하기 위한 약속 ⇒ 클라이언트는 API 통해서 서버에 원하는 것 요청/받음
- SDK (Software Development Kit): 개발을 위해 필요한 도구들을 모아둔 kit
- Package: 누가 만들어둔 코드셋 ⇒ 개발 효율 up
    - 라이브러리:  내가 원하는 규격으로 개발을 하기 위한 도구
    - 프레임워크: 이미 정형화된 규격 내에서 개발을 도와주는 도구
    
    <aside>
    💡
    
    - library와 framework의 구분이 모호할 때도 있다!
    </aside>
    
    - NPM (Node Package Manager): 라이브러리/ 프레임워크의 내용을 저장하는 장소, 여기서 export해서 사용 가능
- IDE(Integrated Development Environment) : 통합 개발 환경 ex)Visual Studio
- 제어 방식
    - CLI(Command Line Interface): 사용자가 명령어 입력을 통해 제어
    - GUI(Graphic User Interface): 사용자가 그래픽 요소로 제어 ex)Internet Explorer
- 도메인:
    1. 개발하는 분야 ex)지도, 쇼핑
    2. IP 주소를 사람이 기억하기 쉽게 만든 인터넷 주소
    - DNS(Domain Name Server): 인터넷 주소(Domain Name)를 쳤을 때 특정 IP 주소로 매핑해준다.
    

## 데이터

- DataBase: 데이터 저장 공간
- 데이터 조회 언어
    - SQL (Structured Query Language): 규격화된 데이터를 조회하기 위한 언어
    - NoSQL (No Structured Query Language): 비규격화된 데이터를 조회하기 위한 언어

## 네트워크 & 배포

- HTTP (Hyper Text Transfer Protocol): HTML을 주고 받을 때 사용하는 통신 규약
    - HTTPS (HTTP Secure): HTTP에 보안 기능 추가
- 서버-클라이언트
    - Server: 클라이언트의 요청에 따라 데이터나 서비스를 제공하는 컴퓨터/프로그램
    - Client: 서버에 서비스 요청을 하는 컴퓨터/사용자
- Deploy(배포): 개발이 완료된 앱을 사용자들이 접근할 수 있는 환경에 올리고 실행하는 과정

## 버젼 관리

- Git: 분산 버전관리 시스템, 수정 사항을 기록 ⇒ 코드의 변경 이력 추적, 동시에 여러명이 코드를 관리하기 용이
    - Commit: 코드의 변경 사항을 기록하는 행위
- Repositoty: 저장소
- DOM (Document Object Model): HTML 문서를 객체로 표현한 모델
- Component: 공통 기능을 재사용하기 위한 단위
- 렌더링: HTML 코드를 화면에 그리는 동작
