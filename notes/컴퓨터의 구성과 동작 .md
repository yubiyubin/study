# 컴퓨터의 구성과 동작

# 1️⃣ 컴퓨터를 한 문장으로 보면

> 컴퓨터는
> 
> 
> **저장장치에 있는 프로그램을 메모리로 올리고,
> CPU가 메모리에 있는 내용을 읽어 처리한 뒤,
> 결과를 다시 메모리나 저장장치로 내보내는 시스템**이다.
> 

이 문장 하나로 전체가 연결됩니다.

---

# 2️⃣ 세 가지 핵심 구성요소

```
[저장장치]  ←→  [메모리]  ←→  [CPU]

```

이 흐름이 **절대 바뀌지 않는 기본 구조**입니다.

---

## ① 저장장치 (SSD / HDD)

### 역할

- **꺼져도 남아 있어야 하는 데이터 보관**
- 운영체제, 프로그램, 파일 저장

### 특징

- 용량 큼
- 속도 느림
- CPU가 **직접 접근 못 함**

👉 CPU는 저장장치를 **바로 실행할 수 없음**

---

## ② 메모리 (RAM)

### 역할

- **지금 실행 중인 프로그램과 데이터 보관**
- CPU와 저장장치 사이의 **작업 공간**

### 특징

- 전원 끄면 데이터 사라짐
- 저장장치보다 빠름
- CPU가 **직접 읽고 씀**

👉 프로그램은 **반드시 RAM에 올라가야 실행 가능**

---

## ③ CPU

### 역할

- **실제 일을 하는 장치**
- 계산, 판단, 제어 담당

### 특징

- 메모리를 읽고 씀
- 저장장치는 직접 만지지 않음
- 모든 행동은 메모리 기반

👉 CPU 입장에서 **세상은 메모리뿐**

---

# 3️⃣ 컴퓨터가 켜질 때 일어나는 큰 흐름

### ① 전원 ON

- CPU는 리셋 상태
- 정해진 주소(펌웨어)부터 실행

---

### ② 부트 프로그램 실행 (저장장치 → 메모리)

```
저장장치(OS)
    ↓ 복사
메모리

```

- 저장장치에 있는 OS 일부를
- 메모리에 올림

👉 이 단계부터 CPU는 **메모리만 보고 동작**

---

### ③ 운영체제 실행

운영체제의 역할:

- 메모리 관리
- CPU 사용 시간 분배
- 저장장치 접근 대행
- 프로그램 실행 관리

👉 CPU는 OS가 시키는 대로만 움직임

---

# 4️⃣ 프로그램 실행의 큰 흐름

```
[저장장치]
   프로그램 파일
        ↓ 로드
[메모리]
   코드 + 데이터
        ↓ 실행
[CPU]
   처리

```

### 정리하면

1. 프로그램 더블클릭
2. OS가 프로그램을 메모리에 복사
3. CPU가 메모리의 코드 실행
4. 결과는 메모리 → 저장장치(파일 저장) or 출력

---

# 5️⃣ 왜 이렇게 나눠져 있을까?

### 저장장치만 쓰면?

- 너무 느림

### 메모리만 쓰면?

- 전원 끄면 전부 사라짐

### CPU가 직접 저장장치 쓰면?

- 구조 복잡
- 속도 병목

👉 그래서 **역할 분담**

| 구성요소 | 담당 |
| --- | --- |
| 저장장치 | 오래 보관 |
| 메모리 | 빠른 작업 공간 |
| CPU | 처리 |

---

# 6️⃣ 비유로 한 번에 이해하기

| 컴퓨터 | 사람 |
| --- | --- |
| 저장장치 | 책장 |
| 메모리 | 책상 위 |
| CPU | 뇌 |
- 책은 책장에서 꺼내서
- 책상 위에 올려야
- 머리로 읽고 생각 가능

👉 책장에서 바로 생각 못 함 = 저장장치 직접 실행 불가

---

# 7️⃣ 한 문장 요약

> 컴퓨터는
> 
> 
> **저장장치 → 메모리 → CPU**
> 
> 이 세 단계를 통해서만 일을 한다.
> 

이 큰 그림이 고정되면,

캐시·가상메모리·프로세스 같은 개념도 자연스럽게 붙습니다.
